HWP / HWPX — 한국어 공문서의 유일한 오픈소스 해법

한글(HWP)은 한국 정부·공공·법률 문서의 표준 포맷이지만, 전 세계 어느 오픈소스 문서 변환기도 네이티브로 지원하지 않습니다. MDM은 HWP 원본 바이너리를 Rust로 직접 파싱하여 텍스트·표·이미지·암호화·법률 문서 구조를 완벽히 복원하는 유일한 오픈소스 프로젝트입니다.

포맷별 실측 점수 — MS 자체 포맷에서도 MDM이 앞섬

각 포맷에서 핵심 피처/엣지 케이스를 측정하여 퍼센트로 환산. 동일 테스트 문서에 대한 변환 성공률 기준.
비교 대상은 Microsoft MarkItDown (pip 최신, MS가 공식 배포하는 Python 변환기). 특히 DOCX · PPTX · XLSX는 Microsoft가 만든 포맷이라 MS의 홈그라운드임에도 MDM이 우세함을 확인할 수 있습니다.

MDM100% (4/4)

발견 & 상호 채택

비교 과정에서 발견한 MarkItDown의 실제 버그들과, MDM이 MarkItDown에서 채용한 로직.

🐛 MS MarkItDown에서 발견한 버그 (4건)

  • DOCX footnote — footnote 있는 문서 변환 시 KeyError 예외 발생, 변환 전체 실패
  • XLSX 파이프 미이스케이프 — 셀 내 |로 GFM 표 구조 파손
  • PPTX 수식 드롭 — python-pptx 경로라 OMML 전처리가 호출되지 않음, 슬라이드 수식 전부 누락
  • HTML mailto/tel 스트립 — URL 스킴 화이트리스트(http/https/file)가 과하게 좁아 정상 이메일/전화 링크까지 제거

✨ MS MarkItDown에서 채용한 로직 (2건)

  • DOCX OMML → LaTeX — MI의 dwml-derived 로직 아이디어를 Rust로 재작성 (core/src/docx/math.rs, MIT attribution)
  • PDF MasterFormat 부분 번호 병합 — .1 / .2 등이 단독 줄로 분리될 때 다음 줄과 병합하는 post-processor

구조 차이

두 프로젝트의 근본적 설계 차이가 결과에 반영됨.

측면 MDM MarkItDown MS
구현 언어 Rust (자체 파서) Python (라이브러리 래퍼)
DOCX 파이프라인 XML → IR → Markdown (직접) DOCX → mammoth(HTML) → markdownify
PDF 접근 Rust + layout 엔진 (헤딩·폰트 감지) pdfplumber + pdfminer (텍스트 덤프)
XLSX 엔진 calamine (XLSX + XLS + ODS) pandas + openpyxl (XLSX만)
포맷 커버리지 철학 동일 포맷 품질 극대화 넓은 포맷 지원 (msg/YouTube/audio/EPUB 등)
한국어 대응 HWP/HWPX 네이티브 + EUC-KR 폴백 HWP 미지원
